# 🕵️ Mystery Partners - Detective Co-op RPG Design Document

> A thrilling detective mystery game where couples work together to solve crimes, analyze clues, and uncover conspiracies

## 🎭 Core Concept

**Mystery Partners** is a cooperative detective RPG where you and your partner become detectives solving interconnected cases in the mysterious city of Shadowbrook. Combining investigation, puzzle-solving, deduction, and story-driven gameplay, this game tests your teamwork and analytical skills.

### Design Pillars

1. **Cooperative Investigation** - Both players must work together to crack cases
2. **Brain Over Brawn** - No combat; solve through deduction and evidence
3. **Choice Consequences** - Your decisions affect the story
4. **Atmospheric Tension** - Mystery and suspense without gore
5. **Story-Driven** - Deep narrative with plot twists

## 👥 Character System

Each player chooses a specialization that gives unique abilities:

## 🎮 Gameplay Mechanics

### 1. **Case Structure**

Each case has 5 phases:

#### Phase 1: Crime Scene Investigation
- Examine the crime scene
- Collect physical evidence
- Take photographs
- Interview witnesses
- Both players explore simultaneously

#### Phase 2: Evidence Analysis
- Analyze collected evidence in the lab
- Run tests (forensics, digital, chemical)
- Connect evidence to suspects
- Update case board

#### Phase 3: Investigation & Interrogation
- Track down suspects
- Conduct interrogations
- Follow leads
- Stake out locations
- Make deductions

#### Phase 4: Connecting the Dots
- Organize evidence on case board
- Create timeline of events
- Identify motive, means, opportunity
- Form hypothesis
- Both players must agree on theory

#### Phase 5: Case Resolution
- Confront the culprit
- Present evidence
- Make accusations
- Witness verdict
- Case scoring (accuracy, time, clues found)

### 2. **Investigation Tools**

#### Evidence Collection
- **Magnifying Glass** - Examine objects closely
- **Camera** - Document crime scenes
- **Evidence Kit** - Collect samples
- **UV Light** - Reveal hidden substances
- **Tape Recorder** - Record testimonies

#### Analysis Tools
- **Microscope** - Analyze tiny evidence
- **Computer Database** - Search records
- **Fingerprint Scanner** - Match prints
- **Chemical Lab** - Test substances
- **Lie Detector** - Aid interrogations

### 3. **Case Board System**

Your shared evidence organization tool:

```
┌────────────────────────────────────┐
│       THE CASE BOARD              │
├────────────────────────────────────┤
│ SUSPECTS:                          │
│  [Photo] [Name] [Motive?]         │
│                                    │
│ EVIDENCE:                          │
│  [Item] → [Links to Suspect]      │
│                                    │
│ TIMELINE:                          │
│  10PM ──── 11PM ──── 12AM         │
│    ↓         ↓         ↓          │
│  [Event]  [Event]  [Event]        │
│                                    │
│ LOCATIONS:                         │
│  [Crime Scene] [Witness Home]     │
│                                    │
│ CONNECTIONS:                       │
│  [Clue] ←→ [Clue] ←→ [Suspect]   │
└────────────────────────────────────┘
```

### 4. **Interrogation System**

**Dialogue Tree with Choices**:
- Question suspects
- Choose approach: Friendly, Neutral, Aggressive
- Present evidence to contradict lies
- Read body language cues
- Both players can input questions

**Lie Detection Mini-Game**:
- Watch for tells: nervous gestures, eye movement
- Behavioral Profiler gets hints
- Press at the right moment to catch lies

### 5. **Deduction Challenges**

**Logic Puzzles**:
- Connect evidence chains
- Eliminate impossible scenarios
- Create motive-means-opportunity diagrams
- Both players solve together

**Timeline Reconstruction**:
- Place events in correct order
- Identify contradictions
- Find alibis and breaks in alibis

## 🛠️ Technical Implementation

### Technology Stack

**Frontend**:
- React + TypeScript (existing)
- React Three Fiber (3D crime scenes)
- Framer Motion (dramatic animations)
- React Flow (case board connections)
- Canvas API (sketching/notes)

**Backend**:
- Supabase PostgreSQL (case data, progress)
- Firebase Realtime DB (shared case board updates)
- Firebase Storage (evidence photos)

### Database Schema

```sql
-- Cases
CREATE TABLE mystery_cases (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  case_number INTEGER NOT NULL,
  title TEXT NOT NULL,
  description TEXT,
  difficulty TEXT CHECK (difficulty IN ('easy', 'medium', 'hard', 'master')),
  story_data JSONB, -- Narrative, locations, NPCs
  evidence_list JSONB, -- All clues
  suspects JSONB, -- Suspect data
  solution JSONB, -- Correct answer
  created_at TIMESTAMP DEFAULT NOW()
);

-- Player Progress
CREATE TABLE detective_progress (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  player_1_email TEXT NOT NULL,
  player_2_email TEXT NOT NULL,
  detective_rank INTEGER DEFAULT 1,
  cases_solved INTEGER DEFAULT 0,
  specialization_1 TEXT, -- Player 1 role
  specialization_2 TEXT, -- Player 2 role
  current_case_id UUID REFERENCES mystery_cases(id),
  achievements JSONB DEFAULT '[]',
  total_playtime INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Case Progress (Active Investigation)
CREATE TABLE active_investigation (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  progress_id UUID REFERENCES detective_progress(id),
  case_id UUID REFERENCES mystery_cases(id),
  evidence_collected JSONB DEFAULT '[]',
  suspects_interrogated JSONB DEFAULT '[]',
  deductions_made JSONB DEFAULT '[]',
  case_board_state JSONB, -- Current board layout
  phase TEXT DEFAULT 'crime_scene',
  started_at TIMESTAMP DEFAULT NOW(),
  completed_at TIMESTAMP,
  score INTEGER,
  time_spent INTEGER
);

-- Completed Cases
CREATE TABLE solved_cases (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  progress_id UUID REFERENCES detective_progress(id),
  case_id UUID REFERENCES mystery_cases(id),
  score INTEGER NOT NULL,
  stars INTEGER CHECK (stars BETWEEN 1 AND 5),
  time_taken INTEGER, -- seconds
  clues_found INTEGER,
  correct_deductions INTEGER,
  solved_at TIMESTAMP DEFAULT NOW()
);

-- Enable RLS
ALTER TABLE detective_progress ENABLE ROW LEVEL SECURITY;
ALTER TABLE active_investigation ENABLE ROW LEVEL SECURITY;
ALTER TABLE solved_cases ENABLE ROW LEVEL SECURITY;

CREATE POLICY "whitelist_only" ON detective_progress FOR ALL USING (is_allowed_user());
CREATE POLICY "whitelist_only" ON active_investigation FOR ALL USING (is_allowed_user());
CREATE POLICY "whitelist_only" ON solved_cases FOR ALL USING (is_allowed_user());
```

## 📝 Sample Case: "The Vanishing Heiress"

**Tier**: Medium  
**Location**: The Heights (Mansion District)  
**Crime**: Missing Person

### Case Brief

> "Victoria Ashford, heiress to the Ashford Fortune, vanished from her locked bedroom last night. No signs of forced entry, no ransom demand. Her fiancé reports she seemed anxious lately. The family wants answers, and fast."

### Crime Scene
- Locked bedroom (window locked from inside)
- Packed suitcase hidden in closet
- Torn letter in fireplace
- Expensive jewelry left behind
- Fresh muddy footprints on balcony

### Suspects
1. **James Sterling** - Fiancé (stood to inherit)
2. **Eleanor Ashford** - Sister (jealous)
3. **Marcus Vale** - Butler (suspicious behavior)
4. **Dr. Hayes** - Family physician (prescribed medication)
5. **Lily Chen** - Best friend (secrets)

### Evidence Trail
- **Forensic Clue**: Mud analysis reveals rare soil
- **Digital Clue**: Deleted emails about affair
- **Behavioral Clue**: Fiancé lying about timeline
- **Physical Clue**: Secret door behind bookshelf

### Solution
Victoria faked her own disappearance to escape arranged marriage. She's hiding at friend's cabin. But who helped her, and why did she leave the footprints?

*(Players must discover twist: The butler helped her escape, motivated by unrequited love and desire to see her happy)*
